mk_d2pt

Perl script: mk_d2pt

NAME
mk_d2pt - raster to vector format conversion for a stack of data files

SYNOPSIS
mk_d2pt <DATA_tab> <pt> <width> <type> <rlks> <azlks> <pdata>

<DATA_tab> (input) list of data files
<plist> (input) point list for sampling (int)
<width> (input) image width (ignored if input is a raster image)
<type> data type of input files:
    0:FCOMPLEX
    1:SCOMPLEX
    2:FLOAT
    3:INT
    4:SHORT
    5:BYTE
    6:SUN/BMP raster
< rlks> (input) range looks of input data relative to point list
<azlks> (input) azimuth looks of input data relative to point list
<pdata> (output) point data stack (same type as f_in)

EXAMPLE
mk_d2pt DATA_tab pt 1200 1 1 0 pdiff

Extracts fcomplex data samples form the set of files listed in the DATA_tab. The locations are located at positions specified in the point list file pt. The data samples are written to the output point data stack pdata. Samples from each file in Data_tab are written to individual layers of pdiff

DESCRIPTION

This script runs the program d2pt for each file list in the DATA_tab. The DATA_tab can be created using the script mk_tab. The use of point data stacks is an important element of the IPTA. Registered data in radar or map coordinates are stored in data stacks with each record corresponding to the values from a single data layer. d2pt is a program that supports sampling input data in natural radar or map coordinates for a number of different data types. The number of looks in range and azimuth refer to a scaling factors between coordinates in the point list pt and the coordinates of the 2d raster data. The rlks parameter refers to the across direction and the azlks parameter to the y coordinate.

The extracted values in binary vector data format are written to the point data stack as successive records. Different data types (fcomplex, scomplex, float, int, short, byte, SUN/BMP raster) are supported. The data type of the input 2-D data and the output point data stack are the same.
