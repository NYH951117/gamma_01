Gamma DIFF/GEO: Reference Manual

Generate ISP and DIFF/GEO parameter files and extract data from CEOS Level 2.1 JERS products

ANSI-C programs: par_JERS_geo.c

NAME
par_JERS_geo - Create ISP image parameter file and DIFF/GEO DEM parameter file for an ellipsoid geocoded JERS SAR image (processing level 2.1)

SYNOPSIS

par_JERS_geo <CEOS_leader> <CEOS_data> <MLI_par> <DEM_par> [MLI]

<CEOS_leader> (input) CEOS leader file provided for Level 2.1 JERS geocoded data (SARL*)
<CEOS_data> (input) JAXA CEOS format Level 2.1 geocoded image data (IMOP*)
<MLI_par> (output) ISP image parameter file (example: yyyymmdd_map.mli.par)
<DEM_par> (output) DIFF/GEO DEM parameter file (example: yyyymmdd_map.dem_par)
[MLI] (output) reformatted JERS geocoded MLI image (example: yyyymmdd_map.mli, enter - for none)

EXAMPLES
par_JERS_geo SARL_01.DAT IMOP_01.DAT 19970506_map.mli.par 19970506_map.dem_par 19970506_map.mli

DESCRIPTION
par_JERS_geo reads a level 2.1 CEOS format geocoded JERS image produced by JAXA EORC. It generates both an ISP image parameter file and DIFF/GEO DEM parameter file. The ISP image parameter file contains the data, polarization, date, and state vector information. The DEM parameter file contains the image dimensions, sample spacing and map projection parameters.

The default map projection for these products is either Universal Transverse Mercator (UTM) or Polar Stereographic (PS). The projection parameters and sample spacing are extracted from the CEOS leader file. Images are projected onto the ellipsoid with 0 m height. Mismatches occurr when the terrain height is above mean sea level or the topography is not flat. Co-registration to an already geocoded image is suggested for futher usage of the image.

The terrain geocoded data are delivered in the form of 16-bit magnitude. The data are converted to 32-bit floating point intensity data. A default calibration factor of -83.6 dB is applied to produce radiometrically calibrated image data. It is possible to generate a SUN raster image of the intensity data using raspwr or ras_dB and then display with geographic information using the disras_dem_par program in the DISP
