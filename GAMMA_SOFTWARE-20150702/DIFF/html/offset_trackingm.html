Gamma DIFF: Reference Manual

offset_trackingm

ANSI-C program: offset_trackingm.c

NAME
offset_trackingm converts range and azimuth offsets files to displacement map.

SYNOPSIS
offset_trackingm <offs> <snr> <MLI_par> <DIFF_par> <coffs_map> [coffsets] [mode] [thres] [poly_flag]

<offs> (input) range and azimuth offset estimates (fcomplex)
<MLI_par> (input) MLI parameter file of reference MLI
<DIFF_par> DIFF/GEO parameter file
<coffs_map> (output) range and azimuth offsets map (fcomplex)
[coffsets] (output) range and azimuth offset estimates and snr values (text file)
[mode] flag indicating displacement mode:
0: displacement in range and azimuth pixels
1: displacement in meters in slant range and azimuth direction
2 (default): displacement in meters in ground range and azimuth direction
[thres] SNR threshold to accept offset value (default from DIFF_par)
[poly_flag] flag indicating if offset polynomials in offset parameter file
shall be subtracted: 0: not subtracted, default=1: subtracted

EXAMPLE
offset_trackingm 1352_1610.offs 1352_1610.snr 1352.MLI.par 1352_1610.diff_par 1352_1610.coffs_map 1352_1610.coffsets 2 6.5 1

Converts the input offset fields (*.offs) to an offset map (*.coffs_map) which has the same dimension as the original reference MLI image.

DESCRIPTION

offset_trackingm converts the input offset fields (*.offs) to an offset map (*.coffs_map) which has the same dimension as the original MLI reference image.

The binary offsets file (*.offs) contains offsets in SAR image pixels (of the reference image) in fcomplex format. The real parts correspond to range offsets and the imaginary parts to azimuth offsets. The positions of the offset estimates are defined by the starting range and azimuth positions and the range and azimuth steps, as documented in the DIFF parameter file.

Only offsets with a quality value (SNR) above an indicated threshold are used. For offsets with an SNR below the threshold and for missing estimates (previously discarded) the offset map values are set to (0.0,0.0).

The culled offsets map (*.coffs_map) is in fcomplex format. The real parts correspond to range offsets and the imaginary parts to azimuth offsets. The file size and positions corespond to those of the reference SAR image.

Different modes are supported. The offsets can be expressed as displacement in range and azimuth pixels (0), or as displacement in meters in slant range and azimuth direction (1), or as displacement in meters in ground range and azimuth direction (2, default mode).

The offset polynomials defined in the offset parameter file can either be subtracted from the offset values (poly_flag=1) or not subtracted (poly_flag=0). The idea of subtracting the offset polynomials is that the polynomials shall represent overall registration offsets, while the offset map shall only contain offsets related to surface displacements.

OPTIONS

None.
